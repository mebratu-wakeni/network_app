# Bulk Product Import Implementation Plan

## Overview
Implement a feature to import multiple products from CSV/Excel files with validation, preview, and batch processing.

## Architecture

### 1. **Components Structure**
```
app/src/components/modules/inventory/
├── ImportProductsModal.js          # Main modal component
└── tabs/
    └── Products.js                  # Update to add import button handler
```

### 2. **Workflow**
```
User clicks "Import Products" 
  → Modal opens
  → User selects file (CSV/Excel)
  → File is parsed
  → Data is validated
  → Preview table shows (with validation status)
  → User reviews and confirms
  → Batch import via API
  → Results summary shown
  → Modal closes / Refresh product list
```

## Implementation Steps

### Step 1: Create ImportProductsModal Component

**File:** `app/src/components/modules/inventory/ImportProductsModal.js`

**Features:**
- File upload input (hidden, triggered by button)
- Support for CSV and Excel files (.csv, .xlsx, .xls)
- File parsing using:
  - **CSV**: PapaParse library (or native FileReader)
  - **Excel**: SheetJS (xlsx) library
- Preview table showing:
  - Row number
  - Product Name
  - Description
  - Category
  - Unit
  - Validation status (✅ Valid / ❌ Error with message)
- Import button (disabled if errors exist)
- Cancel button
- Progress indicator during import
- Results summary after import

**State Management:**
- `file`: Selected file object
- `parsedData`: Array of parsed product objects
- `validationResults`: Array of validation results per row
- `importing`: Boolean for loading state
- `importResults`: Success/error summary after import

### Step 2: File Parsing

**CSV Format Expected:**
```csv
Product Name,Description,Category,Unit
Paracetamol 500mg,Common pain reliever,Regent,Bottle
Bandages,Medical bandages,Supplies,PK
```

**Excel Format:** Same columns as CSV

**Parsing Logic:**
1. Read file using FileReader API
2. Parse based on file extension
3. Map columns to product fields
4. Handle header row (first row may be headers)
5. Return array of product objects

### Step 3: Validation

**Validation Rules:**
- **Product Name**: Required, non-empty string
- **Category**: Required, must exist in system (or allow creation)
- **Unit**: Required, must exist in system (or allow creation)
- **Description**: Optional
- **Product Code**: Auto-generated by system (not in import)

**Validation Function:**
```javascript
function validateProductRow(row, rowIndex, existingCategories, existingUnits) {
  const errors = [];
  
  if (!row.name || row.name.trim() === '') {
    errors.push('Product name is required');
  }
  
  if (!row.category || !existingCategories.includes(row.category)) {
    errors.push('Category is required and must exist');
  }
  
  if (!row.unit || !existingUnits.includes(row.unit)) {
    errors.push('Unit is required and must exist');
  }
  
  return {
    isValid: errors.length === 0,
    errors: errors
  };
}
```

### Step 4: Preview Table

**Display:**
- Table with columns: Row #, Product Name, Description, Category, Unit, Status
- Status column shows:
  - ✅ Green checkmark for valid rows
  - ❌ Red X with error message for invalid rows
- Row highlighting:
  - Valid rows: Light green background
  - Invalid rows: Light red background

### Step 5: Batch Import API

**API Endpoint:** `POST /api/products/bulk-import`

**Request Body:**
```json
{
  "products": [
    {
      "name": "Paracetamol 500mg",
      "description": "Common pain reliever",
      "category": "Regent",
      "unit": "Bottle"
    },
    // ... more products
  ]
}
```

**Response:**
```json
{
  "ok": true,
  "summary": {
    "total": 10,
    "successful": 8,
    "failed": 2
  },
  "results": [
    {
      "index": 0,
      "success": true,
      "product": { "id": 1, "product_code": "PRD001", ... }
    },
    {
      "index": 1,
      "success": false,
      "error": "Category 'InvalidCategory' does not exist"
    }
  ]
}
```

### Step 6: Results Display

**After Import:**
- Show summary card:
  - Total processed: X
  - Successful: Y
  - Failed: Z
- Show detailed results table (optional, collapsible)
- Allow user to:
  - Close modal
  - Download error report (CSV of failed rows)
  - Retry failed imports

## Technical Considerations

### Dependencies Needed

1. **CSV Parsing:**
   ```bash
   npm install papaparse
   ```

2. **Excel Parsing:**
   ```bash
   npm install xlsx
   ```

### File Size Limits
- Maximum file size: 5MB (configurable)
- Maximum rows: 1000 (configurable)
- Show warning for large files

### Error Handling
- File read errors
- Parsing errors (malformed CSV/Excel)
- Validation errors (per row)
- API errors (network, server)
- Partial success handling

### Performance
- Parse file asynchronously
- Show progress for large files
- Batch API calls if needed (e.g., 100 products per request)

## Alternative Approach: Simpler First Version

For MVP, we can start simpler:
1. CSV only (no Excel initially)
2. Basic validation (required fields only)
3. Simple preview (no advanced filtering)
4. Basic error handling
5. No error report download initially

This can be enhanced later based on user feedback.

## Questions to Consider

   - Option C: Show prompt for each new category/unit

2. **How to handle duplicates?**
   - Option A: Skip duplicates (by product name)
   - Option B: Update existing products
   - Option C: Show error for duplicates

3. **Should import be transactional?**

   - Option B: Partial success (import valid rows, skip invalid)
